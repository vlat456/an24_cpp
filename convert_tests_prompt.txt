# Task: Fix All Failing Tests - Add Type Field to Port Initialization

## Problem
After adding PortType field to Port struct, many tests are failing because they create Port objects without specifying the type field:

```cpp
port.ports["v_out"] = Port{PortDirection::Out};  // MISSING type field!
```

All Port initializations need to include type:
```cpp
port.ports["v_out"] = Port{PortDirection::Out, PortType::V};
```

## PortType Reference

Types available (use name from enum, not integer value):
- **PortType::V** - Voltage ports (v, v_in, v_out, v_bus, v_gnd, etc.)
- **PortType::I** - Current ports (i, i_in, i_out, input, etc.)
- **PortType::Bool** - Boolean/logic ports (state, closed, open, control, lamp, cmd, enable, etc.)
- **PortType::RPM** - Rotational speed (rpm, rpm_out, rpm_in, speed, etc.)
- **PortType::Temperature** - Temperature (t, t4, temp, temp_in, temp_out, heat_in, heat_out, etc.)
- **PortType::Pressure** - Pressure (p, pressure, p_out, etc.)
- **PortType::Position** - Position (pos, position)
- **PortType::Any** - Wildcard (any port type)

## Type Inference Rules

Infer type from port name:

1. **Voltage (V)**: contains 'v' (v, v_in, v_out, v_bus, v_start, v_gnd, v_gen, v_ref, k_mod, etc.)

2. **Current (I)**: contains 'i' (i, i_in, i_out, input, primary)

3. **Bool**: contains boolean keywords:
   - state, closed, open, control, lamp, cmd, enable, on, off

4. **RPM**: contains 'rpm' or 'speed'

5. **Temperature**: contains 't' or 'temp' (except when part of other words):
   - t, t4, temp, temp_in, temp_out, heat_in, heat_out

6. **Pressure**: contains 'p' or 'pressure'

7. **Position**: contains 'pos' or 'position'

8. **Any**: default fallback

## Files to Modify

Search for all occurrences of:
```cpp
Port{PortDirection::...}
```

And replace with:
```cpp
Port{PortDirection::..., PortType::...}
```

## Important Notes

- ALWAYS use enum name: `PortType::V`, NOT integer values
- When in doubt, use `PortType::Any` as safe default
- Variable names around the Port creation often contain hints about type
- Look for patterns like: `v_out` → V, `rpm_out` → RPM, `temp` → Temperature
- Don't modify component definitions (components/*.json) - they are already correct
- Only modify TEST and EXAMPLE code that creates Port objects
