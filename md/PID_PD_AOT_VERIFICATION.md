# PID/PD - AOT Codegen Verification

## Overview

Verification that PID and PD components will be properly compiled by the AOT (Ahead-of-Time) code generator.

---

## What the Codegen Does

The AOT code generator (`src/codegen/codegen.cpp`) reads a blueprint and generates:

1. **Header file** (`generated.hpp`) - `Systems` struct with component instances
2. **Source file** (`generated.cpp`) - `solve_electrical()`, `post_step()`, etc.
3. **Port registry** (`port_definitions.h`) - Auto-generated from `components/*.json`

---

## Special Cases (Not Applicable to PID/PD)

| Component | Special Handling | Reason |
|-----------|-----------------|--------|
| `RefNode` | Fixed signals | Alias node, no electrical behavior |
| `Battery` | `inv_internal_r` pre-load | Performance optimization |
| `Bus` / `Voltmeter` | No-op in `solve_electrical` | Pure wiring/measurement only |

PID/PD **don't** need special handling - they use the normal flow.

## Complete AOT Flow for PID

### Input Blueprint

```json
{
  "devices": [
    {
      "name": "voltage_regulator",
      "classname": "PID",
      "params": {
        "Kp": "10.0",
        "Ki": "50.0",
        "Kd": "0.1"
      },
      "ports": {
        "setpoint": {"net": "sp"},
        "feedback": {"net": "vbus"},
        "output": {"net": "ctrl"}
      }
    }
  ]
}
```

### Generated Code (`generated.cpp`)

```cpp
// 1. Instance in Systems struct
PID<AOTProvider> voltage_regulator;

// 2. Explicit template instantiation
template class PID<AOTProvider>;

// 3. Electrical solving (normal flow)
void Systems::solve_electrical(void* state, float dt) {
    voltage_regulator.solve_electrical(*st, dt);  // Stamps 1e-6 conductance
}

// 4. Post-step processing (NOW INCLUDED!)
void Systems::post_step(void* state, float dt) {
    voltage_regulator.post_step(*st, dt);  // Computes PID output
}
```

✅ **Status:** Complete - all 4 steps work correctly

## Testing AOT Compilation

### 1. Create Test Blueprint

```bash
# Create a simple blueprint with PID
cat > test_pid_blueprint.json << 'EOF'
{
  "devices": [
    {
      "name": "battery",
      "classname": "Battery",
      "params": {"v_nominal": "28.0", "internal_r": "0.01"}
    },
    {
      "name": "regulator",
      "classname": "PID",
      "params": {"Kp": "10.0", "Ki": "50.0", "Kd": "0.1"}
    }
  ],
  "connections": []
}
EOF
```

### 2. Run Codegen

```bash
cd build
./bin/an24_codegen \
  --input ../test_pid_blueprint.json \
  --output generated_pid \
  --source-file test_pid_blueprint.json
```

### 3. Verify Generated Files

```bash
# Check that PID is instantiated
grep "PID<AOTProvider>" generated_pid/generated.hpp

# Check that post_step is called
grep "regulator.post_step" generated_pid/generated.cpp

# Compile the generated code
cmake --build . --target codegen_test_pid
```
